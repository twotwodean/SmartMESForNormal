# MES 기능 카탈로그 — 제품소개 자료 분석 (v1.0)

> **문서 목적**: `MES시스템 제품소개/` 캡처 이미지 52장(Frame7 ERP/MES 제품소개)을 판독하여, 웹기반 SmartMES 재구현을 위한 **전체 메뉴·기능 목록**을 정리한다.
> **작성 근거**: 이미지 시각 판독(개요·주요메뉴 10장 직접 판독 + 모듈 상세 42장 6개 그룹 병렬 판독). 화면 스크린샷의 미세 그리드 셀·수치는 저해상도로 다수 판독 불가 → 해당 부분은 "판독 불가"로 명시.
> **분석일**: 2026-07-09
> **다음 단계**: 본 카탈로그 → 구현 기능정의서(우선순위·데이터모델) → 실제 구현.

## 0. 요약 (한눈에)

- **정체**: 이건 스마트팜 MES가 아니라 **범용 이산제조(discrete manufacturing) 기업용 ERP+MES+APS 풀스위트**다. 표지에 `Open Solution ERP 'Frame7'` 표기. APQP·금형/치형구·PVD/CVD·PPM·PUNCH·초중종물·무역원가 등으로 볼 때 **금속가공/자동차 부품 제조사向** 솔루션으로 추정.
- **아키텍처 3축**: `MES(제조실행) + POP(생산시점관리) + QMS(품질경영)` 을 "실시간 정보공유"로 통합. WEB/MES/ERP 서버 분리, ERP 연동 전제.
- **사용자 3계층**: Director PC(경영층·성능분석) / Manager PC(관리자·기준정보·스케줄링) / 현장(KIOSK·PDA·바코드·PLC·센서).
- **규모**: 8대 모듈 · 중분류 약 60개 · 개별 메뉴 **약 500개 이상**.
- **현 SmartMES(팽이버섯)와의 관계**: 도메인이 사실상 다르다. **겹치는 부분은 Lot추적·생산실적·설비(환경)모니터링·품질(부적합)·재고수불·실시간 대시보드(POP)뿐**. 나머지(영업·무역·매입/매출·MRP·APS·금형·회계전표·APQP)는 현 코드에 대응물이 없다.

## 1. 시스템 개요 (이미지 1~4)

### 1.1 MES 주요 모듈 (이미지 1)
01 시스템 구성도 · 02 개발관리 · 03 영업관리 · 04 무역관리 · 05 자재관리 · 06 생산관리 · 07 APS · 08 품질/설비/금형
`Open Solution ERP 'Frame7' – Successful Best ERP Service`

### 1.2 MES 개요 — 3축 개념 (이미지 2)
"제조실행시스템 MES는 주문받은 제품을 최종 생산할 때까지 최적의 생산활동을 하도록 정보를 제공하며, 실시간 데이터로 생산활동을 지시·대응하고 결과를 제공한다."
- **MES**(제조실행 시스템) — 실시간 데이터로 공장 활동 지시·결과 보고
- **POP**(생산시점관리, Point Of Production) — 현장 발생 생산활동 최적화(작업지시·작업실적·품질정보·설비상태)를 실시간 집계·분석
- **QMS**(품질경영시스템) — 사전 예방품질·품질개선·프로세스 표준화
- 3축 중심: **실시간 정보공유**

### 1.3 MES 개념도 — 아키텍처 (이미지 3)
- **사용자/PC 계층**: Director PC(경영층, 성능분석) · Manager PC(관리자, 기준정보·생산계획/스케줄링·생산지표) · 현장 단말
- **서버**: WEB SERVER · MES SERVER · ERP SERVER (Network TCP/IP)
- **MES(생산관리시스템) 내부 블록**: 시스템관리 / 생산·공정관리 / 모니터링 / 설비관리 / 품질관리 / KIOSK / PDA·Barcode
  - 시스템관리: 기준정보·작업장정보·공정정보·설비정보·Routing정보·서열정보·생산이원정보·창고정보
  - 생산/공정관리: 생산계획·작업지시·공정실적관리·생산진척관리·Lot추적정보·작업일보관리·불량실적·외주정보
  - 모니터링: 계획대비실적·공정진척현황·공정별진행현황·설비상태·품질현황·출하현황
  - 설비관리: 실시간설비상태·설비가동관리·설비이력관리·고장현황관리·설비가동율관리
  - 품질관리: 수입검사실적·공정검사실적·출하검사실적·불량현황관리·검사정보관리
  - KIOSK: 작업지시확인·작업실적등록·투입자재관리·불량실적관리·작업진행관리·모니터링
  - PDA/Barcode: 자재입출고·외주입출고·공정이동·제품출고·재고이동·재고조회·바코드출력·Lot Tracking
- **물류/ERP 연동**: 자재입출고·구매발주관리·견적수주관리·영업수금관리·창고 Location·회계관리·인사급여관리·전자결재
- **데이터 수집(Gathering)**: PC 수동입력 / PDA·핸드터미널 / PLC·센서 접점 / 바코드 / 키오스크·산업용PC → 생산실적 실시간 DATA 수집

## 3. 개발관리 (이미지 11~16)

> 신제품 개발·설계정보·BOM·변경이력. 대부분 ERP/PLM 영역 → 초기 제외(Could/Won't).

- **개요(11)**: 신제품 개발 계획수립·프로젝트 관리 + 제품코드/BOM 기준정보·변경이력. 흐름: 프로젝트 등록→PSO 단계→진행등록→진행현황, 모델/사양정보→Model→BOM, 표준원가·품목분류→품목→BOM, 하단 생산계획/MRP 연계.
- **Menu 구조도(12)**: 품목관리·품목분류관리·문서관리·사양관리·모델관리·프로젝트관리·ECO관리·도면관리(§2 Master와 상당 부분 중복).
- **BOM 관리(13)**: E-BOM/M-BOM 정·역·복합 전개, 제품별 재료비 산출.
- **도면/문서관리(14)**: PLM형 도면 리비전·문서 변경이력. 전 모듈 문서 통합(프로젝트·설비·품목·품질·금형/치형구/계측기·총무 문서·사진).
- **APQP(15)**: 신제품 개발 프로젝트 진행현황(진척률 그리드 + 간트차트) + 문서관리 통합, 그리드 행에서 도면 바로 조회.
- **개발투자관리(16)**: 설비·금형·건축물 등 투자 진행·비용 지급 관리, **회계전표 자동발행**(회계시스템 연동).

## 4. 영업관리 (이미지 17~22)

- **개요(17)**: "수주·매출에서 수금관리까지 연동. 제품 흐름 정보 종합관리."
- **영업계획**: 년/3개월/월 계획차수 등록 · 년/3개월/월 모델·제품 판매계획 등록(영업담당자별)·현황·현황(Pivot)

## 10. 설비/금형 관리 (이미지 50~52)

> 설비보전·MTTR/MTBF·전력·금형·치형구. **설비보전은 Should**, 금형/치형구는 Won't.

- **개요(50)**: "설비정보 기반 정비 절차·기준·생명주기·기계이력으로 설비보전 체계 구축. 예방정비기준 근거로 보전작업 자동 스케줄링."
- **설비관리**: 설비 등록·일괄등록·현황 / 설비별 가동시간 현황 / 설비 점검항목·생산품목·예비품 일괄등록
- **설비수리**: 수리의뢰·수리결과 등록 / 수리내역 현황 / **설비 MTTR/MTBF 현황**
- **예비품 관리**: 예비품 등록·현황 / 재고현황 / 입·출고 등록·현황 / 기초재고 등록
- **설비점검**: **점검 스케줄 생성** / 년간 보전 계획·실적 현황 / 점검실적 등록
- **설비등급평가**: 등록·현황
- **소비전력 관리**: 사용기준 조정 / 배부율 등록 / 전력비 등록·산출 / 공장별·부서별 전력비 현황
- **치형구관리**: 치형구 등록·일괄등록·현황 / 생산품목 현황 / 제품별 현황 / 금형 타발현황 / 수리의뢰·결과 등록
- **금형 관리**: 금형 등록·일괄등록·현황 / 금형별 생산품목 / **금형 타수 현황** / 수리의뢰·결과·내역 / 금형 MTTR/MTBF / 금형 이동현황 / 예비품(MRO) 등록·재고·입출고 / 설비 분류코드·위치코드 등록 / 금형 점검항목 등록

## 12. 웹 재구현 우선순위 (MoSCoW 제안)

- **Must (코어 MES)**: 기준정보(품목·공정·작업장·Routing) → 생산계획 → 작업지시 → 생산실적(양품/불량/비가동) → **Lot 추적(계보)** → 재고 수불 → 실시간 모니터링 대시보드(POP)
- **Should**: 품질(인수·공정·출하검사·부적합·PPM·불량코드) → 설비보전(등록·점검 스케줄·수리·MTTR/MTBF) → 근태투입·OEE(설비종합효율)
- **Could**: 영업(수주·출하) → 자재 구매·외주·매입 → SCM(발주·납품·거래명세) → 소요량(MRP) → BOM 전개 → 사양·프로젝트관리
- **Won't (초기 제외 — ERP 영역)**: 무역관리 전체 · APS 자동스케줄러 · 금형/치형구 · 회계전표/원가 · 전자결재 · APQP · FMEA · ISIR · 신뢰성시험 · 수금/여신

## 13. 판독 한계 (신뢰도 고지)

- 개요·메뉴 구조도(트리 텍스트)는 신뢰도 높음(직접/판독 확인).
- **애플리케이션 스크린샷 내부의 그리드 셀 값·컬럼 헤더·입력폼 필드값·차트 수치는 저해상도로 다수 판독 불가** → 화면 "구성/존재"만 기록하고 세부 데이터는 생략.
- APS 스케줄러 조건값·설비 부하율 수치, 품질/설비 Sample 폼 필드값은 특히 신뢰도 낮음(추정 표기).
- 메뉴 트리에 동일 항목 반복(예: 품질(1)·(2) 기준정보 중복)이 있으나 이미지 그대로 전사함.
