# MES 표준 시나리오 기능 갭 분석 v1.0

> 기준: MESA-11 기능 모델 + ISA-95 + 흔한 제조 현장 MES 사용 시나리오. 현재 SmartMES 코드 실측 기반.
> 결론: **비즈니스 흐름(수주→생산→품질→재고→구매/영업→MRP→매출)과 상용화 비기능(동시성·DB·보안·스케일·배포)은 충실**하나, "MES를 ERP와 구분짓는" **현장 실행·설비 자동수집·실시간 가시성** 영역이 보완 필요.

## MESA-11 기능별 현황

| # | 표준 기능 | 현황 | 갭 |
|---|---|---|---|
| 1 | 자원 배정·상태 (RAS) | 설비·작업장 마스터 有, **실시간 상태 無** | Equipment에 상태(RUNNING/IDLE/DOWN) 필드 없음 → 가동상태 미가시 |
| 2 | 상세 스케줄링 (APS) | 생산계획·작업지시 有, **유한능력 스케줄/시퀀싱 無** | Gantt·능력대비부하·순서계획 부재(R4 제외 항목) |

| 4 | 문서 관리 | DocumentRev(메타) 有, **실제 파일/도면 표시 無** | e-작업표준·도면 뷰어·개정 배포 부재 |
| 5 | 데이터 수집 (DAQ) | 수기 POP(ProductionResult) 有, **설비 자동수집 無** | PLC/OPC-UA/센서 연동 없음 — **MES 핵심 차별점 갭** |
| 6 | 노무 관리 | **부재** | 작업자/근무조/자격·숙련도·실적 귀속 모델 없음(ProductionResult에 operator 없음) |
| 7 | 품질 관리 | Inspection·NCR·특채·PPM 有, **SPC 無** | 관리도(X̄-R)·Cp/Cpk·계측데이터 트렌드·CAPA 부재 |

| 9 | 정비 관리 | MaintenanceOrder·Schedule·MTTR/MTBF 有 | **PM 자동 트리거(달력/가동시간 기반)·예비품 무** |
| 10 | 추적성·계보 | Lot + LotGenealogy(정/역방향) 有 ✅ | **리콜 시나리오 UI·시리얼 단위 추적** 얕음 |

## 흔한 현장 시나리오별 갭 (우선순위)

### 🔴 High — MES를 ERP와 구분짓는 핵심
1. **설비 데이터 자동 수집(Level 2 연동)** — 현재 모든 실적이 수기. 표준 MES는 PLC/OPC-UA/센서에서 생산수·가동상태·정지를 자동 취득 → OEE 자동 산출. *(도축 파생은 RFID 스캔으로 이 축을 설계함 — 범용에도 수집 어댑터 필요)*
2. **작업자 현장 실행 흐름 + 실적 귀속** — 작업지시 접수→착수/일시정지/완료, **작업자·설비별 실적**, 스테이션 불량 입력. 현재 ProductionResult에 operator/equipment 없음, 키오스크 흐름 얕음.
3. **실시간 WIP/재공 가시성** — "지금 각 로트가 어느 공정에" + 공정별 재공수량·정체. 현재 재공 현황 화면 없음.
4. **바코드/라벨 + 스캔 기반 이동·소비** — 라벨 발행 없음(기 플래그). 자재 투입/이동을 스캔으로.

### 🟡 Medium — 품질/보전/가시성 심화
5. **SPC(통계적 공정관리)** — 관리도·공정능력(Cpk)·계측 트렌드. 규제·자동차/전자 업종 필수 시나리오.
6. **예방보전(PM) 자동화** — 달력/가동시간 기반 정비지시 자동생성 + 예비품 재고.
7. **Andon/알림** — 라인정지 호출·전광판·에스컬레이션 + 이메일/SMS/푸시 채널(현재 없음).
8. **성과분석 심화** — 정지사유 코드·Pareto, 납기준수율(OTD), 리드타임, 6대손실.
9. **리콜 추적 UI** — 불량 원자재 로트 → 출하 완제품 역추적(계보 로직은 있음, 시나리오 UI 없음).

### 🟢 Lower / 통합·규제
10. **ERP 연동 인터페이스(Level 4)** — 작업지시 수신 / 실적·재고 송신(현재 독립 실행). 표준 MES는 ERP↔현장 중간.
11. **eBR·전자서명(규제 21 CFR Part 11 유형)** — 감사로그는 有, 기록 전자서명 무.
12. **문서/도면 뷰어·e-작업표준 배포**(문서관리 실체화).
13. **능력계획/부하 평준화**(라우팅 stdTime는 있음).

## 권장 로드맵 (범용 SmartMES)
- **P-MES-1**: 실적 귀속 강화 — ProductionResult에 operator/equipment/정지사유코드 + 작업자·근무조 마스터 + 키오스크 실행흐름(착수/완료/불량). → 노무·데이터수집 기반.
- **P-MES-2**: 실시간 WIP 보드(공정별 재공·정체) + 설비 상태(가동/정지) 필드 & 화면 → 가시성·OEE 자동화 토대.
- **P-MES-3**: 라벨/바코드 발행 + 스캔 투입.
- **P-MES-4**: SPC(관리도·Cpk) / PM 자동 트리거 / Andon·알림채널 (택1~다).
- **P-MES-5**: 설비 자동수집 어댑터(OPC-UA/MQTT) — 하드웨어 의존, 별도 스파이크.
- **후속/별도**: APS 스케줄러, ERP 연동, eBR·전자서명.

> 참고: 설비 자동수집·APS·ERP연동은 하드웨어/외부시스템 의존이라 별도 프로젝트/스파이크 권장. 소프트웨어만으로 가장 큰 가치: **P-MES-1~3(현장 실행·WIP·라벨)**.
